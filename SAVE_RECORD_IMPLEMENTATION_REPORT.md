## 实现内容

### 1. 支持的命令格式

#### 保存命令（3种格式）
```
保存 xxx          # 空格分隔
保存：xxx         # 中文冒号分隔
保存:xxx          # 英文冒号分隔
```

#### 记录命令（3种格式）
```
记录 xxx          # 空格分隔
记录：xxx         # 中文冒号分隔
记录:xxx          # 英文冒号分隔
```

**总计：6种命令格式**

---

### 2. 核心功能

#### 功能1：自动创建类别结构
- 自动创建"记录"类别（如果不存在）
- 自动创建"默认"子类别（如果不存在）
- 无需用户手动配置

#### 功能2：智能标题提取
- 使用第一行作为标题
- 如果第一行超过50个字符，自动截断
- 如果没有内容，使用"无标题"

#### 功能3：完整内容保存
- 保存完整的多行内容
- 保存到daily_records表
- 关联到"记录"类别下的"默认"子类别

#### 功能4：聊天记录保存
- 同时保存用户输入到聊天记录
- 保存系统响应到聊天记录
- 便于后续查询和追踪

## 数据库结构

### 创建的类别
```
categories 表
├── id: 自动生成
├── name: '记录'
├── code: 'record'
├── icon: '📝'
└── description: '日常记录'
```

### 创建的子类别
```
subcategories 表
├── id: 自动生成
├── category_id: 指向"记录"类别
├── name: '默认'
├── code: 'default'
└── description: '默认记录'
```

### 保存的记录
```
daily_records 表
├── id: 自动生成
├── user_id: 用户ID
├── title: 标题（第一行或前50个字符）
├── content: 完整内容
├── subcategory_id: 指向"默认"子类别
├── tags: 命令名（'保存'或'记录'）
├── created_at: 创建时间
└── record_date: 记录日期
```

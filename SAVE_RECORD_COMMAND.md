# 保存/记录命令功能实现文档

## 功能概述

实现了快捷保存命令，用户可以使用多种格式快速保存内容到"记录"类别。

---

## 支持的命令格式

### 1. 保存命令
```
保存 xxx          # 空格分隔
保存：xxx         # 中文冒号分隔
保存:xxx          # 英文冒号分隔
```

### 2. 记录命令
```
记录 xxx          # 空格分隔
记录：xxx         # 中文冒号分隔
记录:xxx          # 英文冒号分隔
```

## 使用示例

### 示例1：简单保存
```
用户输入：保存 今天完成了项目A的开发
系统处理：
  1. 检测到"保存"命令
  2. 提取内容："今天完成了项目A的开发"
  3. 创建"记录"类别（如果不存在）
  4. 创建"默认"子类别（如果不存在）

输出：

📄 内容：今天完成了项目A的开发
```

```
  1. 检测到"保存"命令
  2. 提取内容（多行）

输出：

```

```
  1. 检测到"记录"命令
  2. 提取内容："今天的想法和感受"

输出：
```

## 数据库操作

### 创建的类别结构

```
categories (类别表)
├── id: 自动生成
├── name: '记录'
├── code: 'record'
├── icon: '📝'
└── description: '日常记录'

subcategories (子类别表)
├── id: 自动生成
├── category_id: 指向"记录"类别
├── name: '默认'
├── code: 'default'
└── description: '默认记录'

daily_records (日常记录表)
├── id: 自动生成
├── user_id: 用户ID
├── title: 标题（第一行或前50个字符）
├── content: 完整内容
├── subcategory_id: 指向"默认"子类别
├── tags: 命令名（'保存'或'记录'）
├── created_at: 创建时间
└── record_date: 记录日期
```

## 工作流程

```
用户输入
    ↓
检查是否是"保存"或"记录"命令
    ↓
是 → 提取内容
    ↓
    检查"记录"类别是否存在
    ↓
    不存在 → 创建"记录"类别
    ↓
    检查"默认"子类别是否存在
    ↓
    不存在 → 创建"默认"子类别
    ↓
    提取标题（第一行或前50个字符）
    ↓
    保存到daily_records表
    ↓
    保存到聊天记录
    ↓
    返回成功消息
    ↓
否 → 继续其他命令处理
```

## 错误处理

### 错误场景1：未登录

```
输出：❌ 请先登录后再使用此功能
```

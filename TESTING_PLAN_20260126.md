# Flutter 编辑功能修复 - 测试计划

**修复日期**: 2026-01-26
**修复版本**: iOS 29.9MB (build/ios/iphoneos/Runner.app)
**修复内容**: 处理 daily_records 表中空字符串标题的问题

## 📋 问题回顾

### 原始问题
用户反馈：列出"ai助理"列表后，点击"+"号显示"添加ai助理"，但内容实际添加到了"工作"内容中。点击"编辑"按键，打开的编辑栏名称是"ai助理"，但实际显示的却是工作的内容。

### 根本原因
- `daily_records` 表中的 `title` 字段为空字符串 `""` 而非 `null`
- Dart 的 `??` 操作符只处理 `null` 值，不处理空字符串
- 导致编辑功能显示错误的内容

### 实施的修复
在 `_TaskItem` 组件的 5 个位置使用 `.trim().isEmpty` 检查替代 `??` 操作符：

```dart
// 修复前（不处理空字符串）
final title = widget.plan['title']?.toString() ?? widget.plan['content']?.toString() ?? '';

// 修复后（处理空字符串和null）
final titleStr = widget.plan['title']?.toString() ?? '';
final contentStr = widget.plan['content']?.toString() ?? '';
final title = titleStr.trim().isEmpty ? contentStr : titleStr;
```

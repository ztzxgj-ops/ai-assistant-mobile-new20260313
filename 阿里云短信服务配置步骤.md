# 阿里云短信服务配置步骤

## 第一步：开通短信服务

1. 登录阿里云控制台：https://dysms.console.aliyun.com
2. 点击"立即开通"短信服务
3. 同意服务协议

## 第二步：充值

1. 在短信服务控制台，点击左侧"充值"
2. 最低充值100元（可发送约2000条短信）
3. 选择充值金额，完成支付

**价格参考**：
- 国内短信：0.045元/条
- 验证码短信：0.03-0.05元/条

## 第三步：申请短信签名

1. 点击左侧"国内消息" -> "签名管理"
2. 点击"添加签名"
3. 填写信息：
   - **签名名称**：AI助理（或你的应用名称）
   - **签名来源**：选择"企事业单位的全称或简称"
   - **适用场景**：验证码
   - **签名说明**：用于用户注册、登录验证
   - 上传相关证明材料（营业执照等）

4. 提交审核（审核时间：1-2个工作日）

**注意**：签名会显示在短信开头，格式：【AI助理】您的验证码是...

## 第四步：申请短信模板

1. 点击左侧"国内消息" -> "模板管理"
2. 点击"添加模板"
3. 填写注册验证码模板：
   - **模板类型**：验证码
   - **模板名称**：注册验证码
   - **模板内容**：`您的注册验证码是${code}，10分钟内有效，请勿泄露给他人。`
   - **申请说明**：用于用户注册时的身份验证

4. 提交审核（审核时间：1-2个工作日）
5. 审核通过后，会得到一个模板CODE，格式：`SMS_xxxxxxxx`

**可选**：再申请一个密码重置模板：
- **模板内容**：`您正在重置密码，验证码是${code}，10分钟内有效。如非本人操作，请忽略此短信。`

## 第五步：给RAM用户添加短信权限

1. 进入RAM控制台：https://ram.console.aliyun.com/users
2. 找到之前创建的用户：`dm-api-user`
3. 点击"添加权限"
4. 搜索并添加：`AliyunDysmsFullAccess`（短信服务完全访问权限）
5. 确认添加

## 第六步：更新配置文件

审核通过后，需要更新 `aliyun_sms_config.json`：

```json
{
  "sign_name": "AI助理",  // 你申请的签名名称
  "templates": {
    "register": {
      "template_code": "SMS_xxxxxxxx"  // 替换为实际的模板CODE
    }
  }
}
```

## 第七步：测试

配置完成后，运行测试脚本：
```bash
python3 test_sms_send.py
```

## 常见问题

**Q: 签名审核不通过怎么办？**
A: 检查是否提供了有效的证明材料，签名名称是否与证明材料一致。

**Q: 模板审核不通过怎么办？**
A: 确保模板内容符合规范，不包含营销内容，只用于验证码场景。

**Q: 短信发送失败？**
A: 检查：
1. 签名和模板是否审核通过
2. RAM用户是否有短信权限
3. 账户余额是否充足
4. 手机号格式是否正确

**Q: 可以先测试吗？**
A: 可以！不配置短信服务时，系统会自动进入测试模式，验证码会打印在服务器控制台。

# 自动生成备份备注功能说明

## 功能概述

备份系统现在可以根据上次备份后的 git 提交记录自动生成备注，无需手动填写。

## 工作原理

1. **记录 git commit**: 每次备份时，系统会记录当前的 git commit hash
2. **对比变更**: 下次备份时，系统会对比上次备份的 commit 和当前 commit
3. **提取提交信息**: 使用 `git log` 获取两次备份之间的所有提交记录
4. **生成备注**: 自动将提交信息格式化为简洁的备注文本

## 备注格式

### 情况1: 有上次备份记录
```
变更: 提交信息1; 提交信息2; 提交信息3 (共5个提交)
```

### 情况2: 首次备份或无上次记录
```
最近变更: 最近3条提交信息
```

### 情况3: 无新提交
```
无新提交
```

## 使用方法

### 通过 Web 界面备份

1. 访问 http://127.0.0.1:8888
2. 点击"标准备份"或"完整备份"按钮
3. 备份完成后，备注会自动填写

### 通过命令行备份

```bash
# 标准备份
bash backup_standard.sh

# 完整备份
bash backup_complete_system.sh
```

备份完成后，在 Web 界面查看备份历史，备注栏会自动显示 git 变更信息。

## 手动修改备注

如果自动生成的备注不够详细，你仍然可以手动修改：

1. 在 Web 界面找到对应的备份记录
2. 点击"编辑备注"按钮
3. 输入自定义备注
4. 保存

## 技术细节

### 修改的文件

1. **backup_history_manager.py**
   - 添加 `get_current_git_commit()`: 获取当前 git commit hash
   - 添加 `generate_git_notes()`: 根据 git 变更生成备注
   - 修改 `add_backup_record()`: 保存 git commit 到备份记录

2. **backup_web_server.py**
   - 修改备份流程，调用 `generate_git_notes()` 自动生成备注

备份记录新增 `git_commit` 字段：

```json

  "timestamp": "2026-01-14 21:00:00",

  "git_commit": "42b465948a1c2d3e4f5g6h7i8j9k0l1m2n3o4p5q",
  "error": "",
  "files": {
    "code": {
      "name": "ai-assistant-backup-20260114-210000.tar.gz",
      "size": 11869311
    },
    "database": {
      "name": "ai_assistant_db_backup_20260114-210000.sql",
```

## 优势

1. **自动化**: 无需手动记录每次修改内容
2. **准确性**: 直接从 git 提交记录获取，避免遗漏
3. **可追溯**: 每个备份都关联到具体的 git commit
4. **灵活性**: 仍然支持手动修改备注

## 注意事项

1. 需要在 git 仓库中使用此功能
2. 建议在备份前先提交代码到 git
3. 如果不在 git 仓库中，会显示默认备注："备份类型: 标准备份"
4. 旧的备份记录不会自动添加 git_commit 字段，但不影响使用

# 阿里云邮件推送配置详细步骤

## 第一步：开通邮件推送服务

### 1.1 登录阿里云
1. 访问：https://www.aliyun.com
2. 点击右上角"登录"
3. 输入账号密码登录

### 1.2 开通服务
1. 登录后，在顶部搜索框输入：**邮件推送**
2. 点击搜索结果中的"邮件推送 DirectMail"
3. 进入产品页面后，点击"立即开通"
4. 选择计费方式：
   - 推荐选择：**按量付费**（包含每天200封免费额度）
5. 勾选服务协议，点击"立即开通"
6. 开通成功后，点击"管理控制台"进入邮件推送控制台

**控制台直达链接**：https://dm.console.aliyun.com

---

## 第二步：配置发信域名

### 2.1 添加域名
1. 在邮件推送控制台，点击左侧菜单"发信域名"
2. 点击右上角"新建域名"按钮
3. 填写域名信息：
   - **域名**：mail.assistant230.top（或你自己的域名）
   - 建议使用二级域名，如：mail.yourdomain.com
   - 不要使用主域名（yourdomain.com）
4. 点击"确定"

### 2.2 配置DNS记录（重要！）

添加域名后，阿里云会显示需要配置的DNS记录。你需要到你的域名DNS管理处添加这些记录。

#### 如果使用阿里云DNS：

1. 打开新标签页，访问：https://dns.console.aliyun.com
2. 找到你的域名（例如：assistant230.top）
3. 点击"解析设置"
4. 添加以下记录：

**记录1：SPF记录（TXT类型）**
```
记录类型：TXT
主机记录：mail
记录值：v=spf1 include:spf.mxhichina.com -all
TTL：600（10分钟）
```

**记录2：MX记录**
```
记录类型：MX
主机记录：mail
记录值：mxn.mxhichina.com
优先级：10
TTL：600
```

**记录3：CNAME记录（验证用）**
```
记录类型：CNAME
主机记录：（阿里云邮件推送控制台显示的值）
记录值：（阿里云邮件推送控制台显示的值）
TTL：600
```

#### 如果使用其他DNS服务商：

登录你的DNS服务商控制台（如Cloudflare、DNSPod等），按照上述格式添加相同的记录。

### 2.3 验证DNS记录

**等待DNS生效**：
- DNS记录添加后，通常需要10分钟到24小时生效
- 可以使用命令检查：
  ```bash
  ./check_dns.sh
  ```

**在阿里云控制台验证**：
1. 返回邮件推送控制台的"发信域名"页面
2. 找到你添加的域名
3. 点击"验证"按钮
4. 如果DNS已生效，状态会变为"验证通过"

---

## 第三步：创建发信地址

### 3.1 新建发信地址
1. 在邮件推送控制台，点击左侧菜单"发信地址"
2. 点击右上角"新建发信地址"按钮
3. 填写信息：
   - **发信地址**：noreply@mail.assistant230.top
     - 必须使用已验证的域名
     - 建议使用 noreply@ 或 no-reply@ 前缀
   - **发件人名称**：AI个人助理
     - 这个名称会显示在收件人的邮箱中
   - **回信地址**：留空（或填写真实邮箱）
4. 点击"确定"

### 3.2 验证发信地址
- 创建后，发信地址状态应该是"正常"
- 如果需要验证，按照提示完成验证流程

---

## 第四步：创建AccessKey

### 4.1 进入RAM控制台
1. 打开新标签页，访问：https://ram.console.aliyun.com
2. 或在阿里云控制台搜索"RAM访问控制"

### 4.2 创建RAM用户
1. 点击左侧菜单"用户"
2. 点击右上角"创建用户"按钮
3. 填写用户信息：
   - **登录名称**：dm-api-user（或其他名称）
   - **显示名称**：邮件推送API用户
   - **访问方式**：勾选 ✅ **OpenAPI调用访问**
   - 不要勾选"控制台访问"（不需要）
4. 点击"确定"

### 4.3 保存AccessKey（重要！）
创建用户后，会弹出窗口显示AccessKey信息：

```
AccessKey ID: LTAI5tXXXXXXXXXXXXXX
AccessKey Secret: xxxxxxxxxxxxxxxxxxxxxx
```

**⚠️ 非常重要：**
- AccessKey Secret **只显示一次**，关闭窗口后无法再查看
- 请立即复制并保存到安全的地方
- 建议保存到密码管理器或加密文件中

### 4.4 授权用户
1. 在用户列表中，找到刚创建的用户（dm-api-user）
2. 点击用户名进入详情页
3. 点击"权限管理"标签
4. 点击"添加权限"按钮
5. 在搜索框输入：**AliyunDirectMailFullAccess**
6. 勾选这个权限
7. 点击"确定"授权

## ⚠️ 常见问题

### 1. 域名验证失败
- 检查DNS记录是否正确添加
- 等待DNS生效（最多24小时）
- 使用 `./check_dns.sh` 检查DNS记录

### 2. 邮件发送失败
- 检查AccessKey是否正确
- 检查RAM用户是否有权限
- 查看Python控制台错误信息
- 检查发信地址是否正确

### 3. 邮件进入垃圾箱
- 确保SPF记录配置正确
- 避免使用敏感词汇
- 考虑添加DKIM记录（可选）

### 4. 超出免费额度
- 免费版每天200封
- 超出后按0.5元/千封计费
- 可以在控制台查看用量

# App Store上架完整实施计划

**准确北京时间：2026年1月3日**

## 📋 总体计划概览

### 目标
在不超过5个工作日内，将Assistant应用成功上架到App Store。

### 关键里程碑
- 第1天：本地构建、测试、准备证书
- 第2-3天：在App Store Connect创建应用、上传资源
- 第4天：提交审核
- 第5天+：等待审核并根据反馈处理

## 🎯 详细执行计划

### **第1阶段：本地构建和测试（优先级：❌CRITICAL）**

#### 1.1 清理环境并准备构建
```bash
cd /Users/a1-6/Documents/GJ/编程/ai助理new/ai-assistant-mobile
flutter clean
rm -rf ios/Pods ios/Podfile.lock ios/.symlinks
flutter pub get
```

#### 1.2 更新CocoaPods依赖
```bash
cd ios
pod deintegrate
pod install --repo-update
cd ..
```

#### 1.3 构建Release版本
```bash
flutter build ios --release
```

**预期结果**：
- 构建成功，无错误
- 生成Release构建配置

**成功标志**：
- ✅ 命令完成，输出 "flutter build ios --release" 成功

---

### **第2阶段：Xcode中创建Archive（优先级：❌CRITICAL）**

#### 2.1 打开Xcode项目
```bash
open ios/Runner.xcworkspace
```

#### 2.2 在Xcode中配置并创建Archive
**步骤**：
1. 选择 **Runner** 项目和 **Runner** target
2. 点击 **Signing & Capabilities** 选项卡
3. 确认：
   - Team: 已选中（你的Apple开发者账户）
   - Bundle Identifier: `com.gaojun.aiassistant`
   - Automatically manage signing: ✅ 勾选

4. 顶部选择 **Product** → **Scheme** → **Runner**
5. 改为 **Release** 配置
6. 选择真实设备（不是模拟器）
7. **Product** → **Build** （验证无错误）
8. **Product** → **Archive**

**预期时间**：5-10分钟

**成功标志**：
- ✅ Archive 完成
- ✅ 在 Organizer 中可看到新Archive

---

### **第3阶段：导出IPA并上传（优先级：❌CRITICAL）**

#### 3.1 在Organizer中导出
**步骤**：
1. **Window** → **Organizer**（或 **Product** → **Organizer**）
2. 找到刚创建的Archive
3. 点击 **Distribute App**
4. 选择 **App Store Connect** → **Upload**
5. 选择Team
6. 自动完成签名
7. 确认上传完成

**预期时间**：3-5分钟（上传速度取决于网络）

**成功标志**：
- ✅ 显示"Upload Successful"消息
- ✅ 可在App Store Connect中看到新构建

---

### **第4阶段：在App Store Connect中创建应用（优先级：❌CRITICAL）**

#### 4.1 登录App Store Connect
访问：https://appstoreconnect.apple.com

#### 4.2 创建新应用
**步骤**：
1. 点击左侧 **我的App**
2. 点击右上角 **+** → **新建App**
3. 选择平台：**iOS**
4. 填写表单：
   ```
   应用名称：Assistant
   套件ID：com.gaojun.aiassistant
   SKU：assistant_2026_01
   用户访问权限：完全访问
   ```
5. 点击 **创建**

**成功标志**：
- ✅ 应用已创建
- ✅ 可在Dashboard中看到Assistant应用

### **第11阶段：提交审核（优先级：❌CRITICAL）**

#### 11.1 提交
1. 点击右上角 **提交审核** 按钮
2. 如果出现IDFA选项，选择：
   - ❌ 跟踪（没有第三方跟踪）
   - ❌ 数据链接到用户（不跟踪用户行为）
   - ❌ 第三方广告（没有广告）
3. 确认选择
4. 点击 **提交**

**成功标志**：
- ✅ 显示"提交成功"消息
- ✅ 应用状态变为"等待审核"

---

### **第12阶段：后续跟踪（优先级：🟡HIGH）**

#### 12.1 监控审核进度
- 定期检查App Store Connect中的审核状态
- 预期审核时间：1-5个工作日
- 检查邮箱，可能收到Apple的反馈

#### 12.2 准备应对反馈
如果Apple拒绝应用，常见原因：
- 隐私政策缺失或无效
- 功能不完整
- 后端服务不稳定
- 权限申请不必要
- 内容违规

#### 12.3 审核通过后发布
1. 在App Store Connect中看到"Ready for Release"状态
2. 点击 **发布此版本**
3. 应用将在App Store上线（通常几小时内）

## 🔧 集成步骤

### 第一步：更新verification_service.py

将新的SMSService实现集成到现有代码中：

**选项A：替换SMSService类（推荐）**

1. 打开 `verification_service.py`
2. 找到 `class SMSService:` (第150行左右)
3. 删除整个SMSService类（第150-178行）
4. 从 `aliyun_sms_service.py` 复制 `AliyunSMSService` 类
5. 重命名为 `SMSService`（保持类名不变）

**选项B：导入新的实现**

在 `verification_service.py` 顶部添加：
```python
from aliyun_sms_service import AliyunSMSService as SMSService
```

然后注释掉原来的SMSService类定义。

## 🚀 配置阿里云短信服务（生产模式）

### 第一步：开通服务

1. 访问：https://dysms.console.aliyun.com
2. 点击"立即开通"
3. 充值（建议100元起）

### 第二步：申请签名和模板

**申请签名：**
- 签名名称：AI助理
- 签名来源：网站
- 签名场景：验证码
- 审核时间：1-2个工作日

**申请模板：**

注册验证码模板：
```
您的注册验证码是${code}，10分钟内有效，请勿泄露给他人。
```

密码重置验证码模板：
```
您正在重置密码，验证码是${code}，10分钟内有效。如非本人操作，请忽略此短信。
```

审核时间：1-2个工作日

### 第三步：添加权限

1. 访问RAM控制台：https://ram.console.aliyun.com
2. 找到用户：dm-api-user
3. 添加权限：**AliyunDysmsFullAccess**

### 第四步：更新配置文件

编辑 `aliyun_sms_config.json`：

```json
{
  "access_key_id": "你的AccessKey ID（与邮件服务相同）",
  "access_key_secret": "你的AccessKey Secret",
  "region": "cn-hangzhou",
  "sign_name": "AI助理",
  "templates": {
    "register": {
      "template_code": "SMS_XXXXXXX"
    },
    "reset_password": {
      "template_code": "SMS_YYYYYYY"
    }
  }
}
```

### 第五步：测试发送

```bash
python3 test_sms_send.py
```

输入真实手机号，检查是否收到短信。

## 📊 API使用示例

### 发送短信验证码

```bash
curl -X POST http://localhost:8000/api/verification/send-code \
  -H "Content-Type: application/json" \
  -d '{
    "contact_type": "phone",
    "contact_value": "13800138000",
    "code_type": "register"
  }'
```

### 验证验证码

```bash
curl -X POST http://localhost:8000/api/verification/verify-code \
  -H "Content-Type: application/json" \
  -d '{
    "contact_type": "phone",
    "contact_value": "13800138000",
    "code": "123456",
    "code_type": "register"
  }'
```

### 手机号注册

```bash
curl -X POST http://localhost:8000/api/auth/register-with-verification \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "password123",
    "phone": "13800138000",
    "verification_code": "123456"
  }'
```

## ⚠️ 注意事项

### 1. 防刷机制

系统已实现：
- ✅ 60秒内同一手机号只能发送1次
- ✅ 24小时内同一手机号最多10次
- ✅ 验证码有效期10分钟
- ✅ 最多5次验证失败

### 2. 安全建议

- ✅ 不要在前端暴露AccessKey
- ✅ 使用HTTPS传输验证码
- ✅ 记录发送日志，便于排查
- ✅ 监控发送量，防止恶意刷短信

### 3. 用户体验

- ✅ 提供"重新发送"功能（60秒后）
- ✅ 显示倒计时
- ✅ 提示验证码有效期
- ✅ 发送失败时给出明确提示

## 🔍 故障排查

### 问题1：短信收不到

**检查清单：**
- [ ] 手机号格式是否正确（11位数字）
- [ ] 签名和模板是否审核通过
- [ ] AccessKey权限是否正确
- [ ] 账户余额是否充足
- [ ] 是否被运营商拦截
- [ ] 是否被手机安全软件拦截

### 问题2：发送失败

**常见错误码：**
- `isv.BUSINESS_LIMIT_CONTROL`: 触发流控（发送过于频繁）
- `isv.MOBILE_NUMBER_ILLEGAL`: 手机号格式错误
- `isv.TEMPLATE_MISSING_PARAMETERS`: 模板参数缺失
- `isv.AMOUNT_NOT_ENOUGH`: 余额不足

**解决方法：**
1. 查看控制台错误日志
2. 检查错误码对应的原因
3. 调整发送频率或充值

### 问题3：测试模式无法切换到生产模式

**检查：**
- [ ] `aliyun_sms_config.json` 中的AccessKey是否填写正确
- [ ] AccessKey是否包含"填写"等占位符
- [ ] 是否重启了服务器

# 阿里云短信服务配置指南

## 📋 概述

本指南将帮助你配置阿里云短信服务，实现手机验证码功能。

## 💰 费用说明

- **国内短信**：0.045元/条（充值后价格）
- **验证码短信**：通常每条0.03-0.05元
- **最低充值**：100元起
- **建议充值**：100-200元（可发送2000-4000条短信）

## 🚀 第一步：开通短信服务

### 1.1 访问短信服务控制台

访问：https://dysms.console.aliyun.com

### 1.2 开通服务

1. 如果未开通，点击"立即开通"
2. 阅读并同意服务协议
3. 点击"立即开通"

### 1.3 充值

1. 在控制台点击"充值"
2. 选择充值金额（建议100元起）
3. 完成支付

---

## 📝 第二步：申请短信签名

### 2.1 什么是短信签名

短信签名是短信开头的【】内容，例如：
```
【AI助理】您的验证码是123456，10分钟内有效。
```
其中【AI助理】就是短信签名。

### 2.2 创建签名

1. 在短信服务控制台，点击左侧"国内消息" → "签名管理"
2. 点击"添加签名"
3. 填写信息：
   ```
   签名名称：AI助理
   签名来源：网站
   签名场景：验证码
   签名说明：用于AI个人助理系统的用户验证
   ```
4. 上传证明材料（如果需要）：
   - 网站备案截图
   - 或营业执照
   - 或其他证明材料
5. 点击"提交"

### 2.3 等待审核

- 审核时间：1-2个工作日
- 审核通过后，签名状态会变为"审核通过"

---

## 📄 第三步：申请短信模板

### 3.1 什么是短信模板

短信模板是短信的内容格式，例如：
```
您的验证码是${code}，${time}分钟内有效。
```

### 3.2 创建验证码模板

1. 在短信服务控制台，点击"模板管理"
2. 点击"添加模板"
3. 填写信息：

**注册验证码模板：**
```
模板名称：用户注册验证码
模板类型：验证码
模板内容：您的注册验证码是${code}，10分钟内有效，请勿泄露给他人。
```

**密码重置验证码模板：**
```
模板名称：密码重置验证码
模板类型：验证码
模板内容：您正在重置密码，验证码是${code}，10分钟内有效。如非本人操作，请忽略此短信。
```

4. 点击"提交"

### 3.3 等待审核

- 审核时间：1-2个工作日
- 审核通过后，会获得模板CODE（如：SMS_123456789）

---

## 🔑 第四步：使用AccessKey

短信服务可以使用邮件服务的同一个AccessKey：

- **AccessKey ID**: 你已经创建的dm-api-user的AccessKey
- **AccessKey Secret**: 已保存的Secret

**权限要求：**
需要给RAM用户添加短信服务权限：

1. 访问RAM控制台：https://ram.console.aliyun.com
2. 找到用户：dm-api-user
3. 点击"添加权限"
4. 搜索并选择：**AliyunDysmsFullAccess**
5. 点击"确定"授权

---

## 📋 第五步：记录配置信息

审核通过后，记录以下信息：

```
短信签名：AI助理
注册验证码模板CODE：SMS_XXXXXXX
密码重置验证码模板CODE：SMS_YYYYYYY
AccessKey ID：（已有）
AccessKey Secret：（已有）
```

## 🔧 第六步：创建配置文件

在项目目录创建 `aliyun_sms_config.json`：

```json
{
  "_comment": "阿里云短信服务配置文件",

  "access_key_id": "你的AccessKey ID",
  "access_key_secret": "你的AccessKey Secret",
  "region": "cn-hangzhou",

  "sign_name": "AI助理",
  "_sign_name_help": "短信签名，必须是已审核通过的签名",

  "templates": {
    "register": {
      "template_code": "SMS_XXXXXXX",
      "description": "注册验证码模板"
    },
    "reset_password": {
      "template_code": "SMS_YYYYYYY",
      "description": "密码重置验证码模板"
    }
  },

  "_testing_mode": {
    "_comment": "测试模式说明",
    "_note": "如果access_key_id为空或占位符，将进入测试模式",
    "_test_behavior": "验证码会打印到控制台，不会发送真实短信"
  }
}
```

## ⚠️ 常见问题

### 1. 签名审核不通过

**原因：**
- 签名名称不符合规范
- 缺少证明材料
- 签名与业务不符

**解决：**
- 查看审核意见
- 补充证明材料
- 修改签名后重新提交

### 2. 模板审核不通过

**原因：**
- 模板内容包含敏感词
- 模板格式不规范
- 变量使用不当

**解决：**
- 查看审核意见
- 修改模板内容
- 确保使用${变量名}格式

### 3. 发送失败

**常见错误码：**
- `isv.BUSINESS_LIMIT_CONTROL`: 触发流控限制（发送过于频繁）
- `isv.MOBILE_NUMBER_ILLEGAL`: 手机号格式错误
- `isv.TEMPLATE_MISSING_PARAMETERS`: 模板参数缺失
- `isv.AMOUNT_NOT_ENOUGH`: 账户余额不足

**解决：**
- 检查错误码对应的原因
- 调整发送频率
- 验证手机号格式
- 充值账户余额

### 4. 短信收不到

**可能原因：**
- 手机号被运营商拦截
- 短信被手机安全软件拦截
- 手机信号问题
- 短信网关延迟

**解决：**
- 检查手机拦截设置
- 尝试其他手机号
- 等待几分钟后重试
- 查看短信发送日志

## 📊 监控和管理

### 查看发送统计

访问：https://dysms.console.aliyun.com

- 点击"数据统计"查看发送量、成功率等
- 点击"发送记录"查看详细的短信发送记录

### 设置发送限制

为了防止恶意刷短信，建议设置：

1. **频率限制**：同一手机号60秒内只能发送1次
2. **日限制**：同一手机号24小时内最多10次
3. **IP限制**：同一IP每小时最多发送50次

这些限制已经在 `VerificationManager` 类中实现。

## 💡 最佳实践

1. **签名规范**：使用简短、易识别的签名
2. **模板规范**：内容简洁明了，包含必要信息
3. **防刷机制**：实现频率限制和日限制
4. **错误处理**：记录发送失败的原因，便于排查
5. **成本控制**：监控发送量，设置预警
6. **用户体验**：提供"重新发送"功能，但有时间限制

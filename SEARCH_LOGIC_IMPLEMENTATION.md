# 智能搜索逻辑实现文档

## 概述

根据用户需求，实现了一套完整的智能搜索逻辑，区分"X"和"X相关"两种查询方式，提供不同的搜索范围和结果展示。

---

## 搜索逻辑规则

### 1. 查询"X"（不带"相关"）

**搜索范围：**
- 类别（categories）
- 子类别（subcategories）
- 关键词（search_keywords）

**返回结果：**
- 精确匹配的分类信息
- 该分类下的记录列表

**无结果提示：**
```
没有记录'X'信息
```

**示例：**
```
用户查询："ai"
系统搜索：categories、subcategories 表中是否有"ai"相关的分类
返回：如果找到"ai助理"子类别，显示该子类别下的所有记录
```

---

### 2. 查询"X相关"（带"相关"）

**搜索范围：**
- 聊天记录（messages）
- 日常记录（daily_records）
- 留言墙（guestbook_messages）
- 工作计划（work_plans）
- 提醒（reminders）

**返回结果：**
- 所有包含"X"的相关信息
- 按数据来源分组显示
- 显示完整的搜索结果（无数量限制）

**无结果提示：**
```
没有记录'X'相关信息
```

**示例：**
```
用户查询："留言墙相关"
系统搜索：
  1. messages 表中包含"留言墙"的聊天记录
  2. daily_records 表中包含"留言墙"的日常记录
  3. guestbook_messages 表中的留言墙数据
  4. work_plans 表中包含"留言墙"的工作计划
返回：所有找到的相关信息，按类型分组显示
```

---

## 实现细节

### 核心方法

#### 1. `_fuzzy_match_subcategory(user_message, user_id)`

**功能：** 处理"X相关"查询

**逻辑流程：**
```
1. 提取"X相关"中的关键词X
2. 在subcategories表中查找包含X的子类别
3. 如果找到子类别：
   - 返回子类别名称（字符串）
   - 递归调用chat()处理该子类别
4. 如果未找到子类别：
   - 调用_comprehensive_search_related()进行全面搜索
   - 返回搜索结果（字典格式）
```

**返回值：**
- 字符串：子类别名称（如"ai助理"）
- 字典：全面搜索结果（包含response、is_comprehensive_search等字段）
- None：无匹配

#### 2. `_comprehensive_search_related(keyword, user_id)`

**功能：** 全面搜索所有数据表

**搜索流程：**
```
1. 搜索messages表（聊天记录）
2. 搜索daily_records表（日常记录）
3. 搜索guestbook_messages表（留言墙）
4. 搜索work_plans表（工作计划）
5. 合并所有结果
```

**返回值：**
```python
[
    {
        'type': '聊天记录',
        'content': '...',
        'timestamp': '2026-01-20 15:58',
        'source': 'messages'
    },
    {
        'type': '留言墙',
        'content': '...',
        'timestamp': '2026-01-19 11:36',
        'source': 'guestbook'
    },
    ...
]
```

#### 3. `_format_comprehensive_search_results(keyword, results)`

**功能：** 格式化搜索结果

**输出格式：**
```
🔍 找到 X 条与'留言墙'相关的信息：

📌 聊天记录（共X条）：
  1. 内容摘要...
     [2026-01-20]
  2. 内容摘要...
     [2026-01-20]

📌 日常记录（共X条）：
  3. 内容摘要...
     [2026-01-20]

📌 留言墙（共X条）：
  4. 内容摘要...
     [2026-01-19]
```

**无结果输出：**
```
没有记录'留言墙'相关信息
```

## 性能考虑

1. **搜索范围限制：** 每个表的搜索都有LIMIT限制（20条），防止过大的结果集
2. **关键词提取：** 使用正则表达式快速提取关键词
3. **缓存机制：** 可以考虑缓存常见查询结果
4. **数据库索引：** 确保搜索字段有适当的索引
